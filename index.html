Assingability error

In TypeScript, an assignability error occurs when you try to assign a value to a variable or property that is not compatible with the type expected by that variable or property. Simply put, it means that the type of the value you are trying to assign does not match the type that TypeScript expects.

Strong typing

Strong typing in TypeScript means that once a variable or a property is declared to be of a certain type, it must always hold values of that type. The compiler enforces these type rules, helping to catch errors at compile time rather than at runtime.
In simpler terms, it means that TypeScript checks and ensures that values match the expected types throughout your code.

Let , const and var

In TypeScript, let, const, and var are used to declare variables, each with different rules and behaviors.
let and const are block-scoped; let can be reassigned, const cannot.
var is function-scoped and can be reassigned and redeclared.

Let

Used to declare a variable that can be reassigned later.
The variable is block-scoped, meaning it only exists within the block (e.g., within curly braces {}) where it's defined.
You cannot declare same name variable with let.

Const

Used to declare a variable that cannot be reassigned after its initial assignment.
The variable is block-scoped, similar to let.

Var

Used to declare a variable that can be reassigned.
The variable is function-scoped, meaning it exists within the function where it's defined or globally if not inside a function.
Unlike let and const, var variables can be redeclared and do not respect block scope.

Union Types

In TypeScript, a union type lets a variable hold more than one type of value. It means the variable can be one type or another type.

Literal Type

Literal types are particularly useful when you want to restrict variables to a finite set of known values, making your code more explicit and easier to reason about.

Objects

In TypeScript, an object is a collection of key-value pairs, where each key is a string (or symbol) and each value can be any type, such as a number, string, or even another object.

Object Alias

In TypeScript, an object alias is a custom name you give to a particular shape or structure of an object. This lets you define and reuse complex object types using a single, easy-to-remember name.

Structural Typing

In TypeScript, structural typing means that the type system checks if objects are compatible based on their structure or shape, not just their declared type. If two objects have the same properties with the same types, TypeScript considers them compatible.

Nested Objects

In TypeScript, nested objects are objects inside other objects. This means a property of one object is itself another object.

Intersection

In TypeScript, intersection types allow you to combine multiple types into one.This combined type includes all the properties and methods of the types being merged.

Unknown

In TypeScript, `unknown` is like `any` because it can store values of any type. But unlike `any`, you have to check or confirm its type before using it. This makes sure that your code remains type-safe when handling values whose types aren't known until runtime.

Any

The any type in TypeScript allows you to assign any kind of value to a variable or expression, bypassing TypeScript's type checking.

explict casting

Explicit type casting in TypeScript, also known as type assertion, is a way to tell TypeScript that you know more about the type of a value than TypeScript does initially. It allows you to treat a value as a different type.

Const enum

A `const enum` in TypeScript allows you to use enums more efficiently by inlining their values at compile-time. This reduces the size of the generated JavaScript code, making it especially useful for performance-sensitive applications or when you need to keep your code small.

Functions

Functions in TypeScript are blocks of reusable code that perform a specific task when called.

Rest parameter

The rest parameter in TypeScript allows a function to accept an indefinite number of arguments as an array.

ConCurrency

Concurrency in TypeScript (or any programming language) refers to the ability to perform multiple operations at the same time, rather than waiting for one to complete before starting the next. This is particularly useful when dealing with tasks that can take some time, like fetching data from a server or reading files.

Promises

A Promise in TypeScript is an object that represents a task that will be completed in the future. It can either succeed or fail. When the task succeeds, the Promise is "resolved" with a result. When it fails, the Promise is "rejected" with an error.

Callback function

A callback function in TypeScript is a function that you pass as an argument to another function, and it gets called (or "called back") after the first function completes its task.

Asynchronous

Asynchronous programming in TypeScript is a way to write code that doesn't block or wait for tasks to complete. Instead, tasks run in the background, and the code moves on to the next task This is particularly useful when dealing with tasks that can take some time, like fetching data from a server or reading files.
